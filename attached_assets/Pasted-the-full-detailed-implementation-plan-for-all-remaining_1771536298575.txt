Pattern to implement:

const allocateInventoryStep = createStep(
  "allocate-inventory-step",
  async (input, { container }) => {
    const inventoryModule = container.resolve("inventory") as any
    const allocations = await inventoryModule.createReservationItems(input.items)
    // Return data + compensation context
    return new StepResponse(
      { allocations },
      { allocationIds: allocations.map(a => a.id) } // compensation data
    )
  },
  // COMPENSATION FUNCTION — runs if a later step fails
  async (compensationData, { container }) => {
    const inventoryModule = container.resolve("inventory") as any
    await inventoryModule.deleteReservationItems(compensationData.allocationIds)
  }
)
Acceptance criteria: Every step that creates, updates, or allocates resources has a compensation function. Compensations are idempotent (safe to re-run).

Sprint 4: Middleware Hardening — ~1 week
4A. Centralized Store Auth Middleware
Instead of checking auth in every route handler, add middleware patterns:

// In middlewares.ts — protect all store mutation routes that need auth
{
  matcher: "/store/subscriptions/**",
  method: ["POST", "PUT", "PATCH", "DELETE"],
  middlewares: [requireCustomerAuth],
},
{
  matcher: "/store/wallet/**",
  method: ["POST", "PUT", "PATCH", "DELETE"],
  middlewares: [requireCustomerAuth],
},
// ... repeat for all protected routes
4B. Rate Limiting
Add rate limiting for store POST endpoints to prevent abuse:

General store POST: 30 requests/minute per IP
Subscription checkout: 5 requests/minute per customer
Wallet operations: 10 requests/minute per customer
Newsletter signup: 3 requests/minute per IP
4C. Request Logging
Add structured request/response logging middleware:

Log method, path, response status, duration
Redact sensitive fields (passwords, tokens, card numbers)
Include correlation IDs for tracing
Dependencies: Sprint 1 (auth + validation) should be done first so middleware doesn't break routes that aren't yet handling validation correctly.

Acceptance criteria: middlewares.ts covers all protected routes. Rate limiting active on POST endpoints. Request logs available for debugging.
